Info

Submitted intake details

The bio/intake details from the first care or provider form are kept inside each request card under “Show submitted intake details.” They stay hidden by default so the dashboard does not expose sensitive details at a glance.

Those details decrypt only on browsers that still have the local request key.

Security

Encryption and recovery

Current care requests use browser-held encryption keys. Password reset can restore account access in a future account system, but it should not unlock old encrypted care details unless a recovery key or trusted-device recovery is configured.

Do not store PHI in email, URLs, payment metadata, analytics, or support tickets.

A local lock helps when someone else uses this device. It does not replace encryption or device-level security.
